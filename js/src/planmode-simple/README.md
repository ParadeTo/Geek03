# 简单计划模式 (Plan Mode - Simple)

## 概述

简单计划模式是一种"先计划，后执行"的 Agent 设计模式。与 ReAct 模式边思考边行动不同，计划模式会先制定完整的执行计划，然后按计划逐步执行。

## 核心思想

1. **分离规划与执行**：先用 LLM 生成完整计划，再按计划执行
2. **结构化流程**：计划 → 执行 → 工具调用 → 循环执行
3. **清晰可控**：每一步都在预定计划内，便于监控和调试

## 工作流程

```
用户问题 → 生成计划 → 执行步骤 → 调用工具 → 判断 → 循环或结束
                                      ↑              ↓
                                      └──────────────┘
```

## 运行

```bash
npm run plan:simple
```

## 与 ReAct 的区别

| 特性 | ReAct | 简单计划模式 |
|-----|-------|-------------|
| 决策时机 | 边执行边决策 | 先规划再执行 |
| 灵活性 | 高（可随时调整） | 低（按固定计划） |
| 可预测性 | 低 | 高 |
| 适用场景 | 探索性任务 | 结构化任务 |

## 示例

**输入：** "茅台和青岛啤酒哪个贵？"

**生成的计划：**
```
1. 获取贵州茅台的收盘价
2. 获取青岛啤酒的收盘价
3. 比较两者价格，得出结论
```

**执行过程：**
- 调用工具获取茅台价格: 1488.21
- 调用工具获取青岛啤酒价格: 67.92
- 输出结论: 茅台更贵
